# ASPPNeXt Fine-Tuning Recommendations for >95.18% Accuracy

## 1. Learning Rate Scheduler
Add a scheduler for more effective convergence. CosineAnnealingLR or ReduceLROnPlateau work well.

```python
# After optimizer definition:
from torch.optim.lr_scheduler import CosineAnnealingLR, ReduceLROnPlateau

# Option 1: Cosine Annealing
scheduler = CosineAnnealingLR(optimizer, T_max=10, eta_min=1e-7)
# Option 2: Reduce LR on Plateau
scheduler = ReduceLROnPlateau(optimizer, mode='min', factor=0.5, patience=4, min_lr=1e-7)
```
Call `scheduler.step(val_loss)` at the end of each epoch.

## 4. Mix Loss Functions
Combine Focal Tversky with Dice Loss or BCE for better gradient signals.

```python
class ComboLoss(nn.Module):
    def __init__(self, alpha=0.5):
        super().__init__()
        self.ft = FocalTverskyLoss()
        self.dice = DiceLoss()
        self.alpha = alpha
    def forward(self, pred, target):
        return self.alpha * self.ft(pred, target) + (1 - self.alpha) * self.dice(pred, target)
```

## 5. Early Stopping
Avoid overfitting by stopping when validation loss plateaus.

```python
# Pseudocode
best_loss = float('inf')
patience = 10
wait = 0
for epoch in range(num_epochs):
    ...
    if val_loss < best_loss:
        best_loss = val_loss
        wait = 0
    else:
        wait += 1
    if wait > patience:
        print("Early stopping!")
        break
```

## 8. Use Stochastic Weight Averaging (SWA)
Averaging weights across epochs can yield better generalization.

```python
from torch.optim.swa_utils import AveragedModel, SWALR

swa_model = AveragedModel(model)
swa_scheduler = SWALR(optimizer, swa_lr=3e-5)
# During training (after some epochs):
swa_model.update_parameters(model)
swa_scheduler.step()
# At end, swap model weights with swa_model
model.load_state_dict(swa_model.state_dict())
```
